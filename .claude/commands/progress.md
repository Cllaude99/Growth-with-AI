# 학습 진행 현황

`progress/log.md` 파일을 읽고 학습자의 진행 현황을 보여주세요.

## 표시할 내용

### 1. 전체 통계
- 총 학습 세션 수
- 다룬 토픽 수 / 전체 토픽 수
- 연속 학습일 수
- 마지막 학습 날짜

### 2. 카테고리별 커버리지 맵
각 카테고리(자료구조 & 알고리즘, 운영체제, 네트워크, 데이터베이스, 시스템 설계, 소프트웨어 공학, 웹 & 프론트엔드)별로:
- 완료한 토픽 / 전체 토픽
- 진행률 바 시각화 (예: `[████░░░░░░] 40%`)
- 각 토픽의 상태(`-`/`학습중`/`통과`), 도달 난이도, 마지막 학습일

### 3. 최근 세션 (최근 5개)
- 날짜, 토픽, 모드(CS학습/디버깅), 핵심 포인트

### 4. 추천 다음 토픽
progress/log.md의 기록을 분석하여:
- **검증 재도전**: "학습중" 상태인 토픽 (검증 미통과, 우선 추천)
- **복습 필요**: 7일 이상 지난 토픽 중 난이도가 낮았던 것
- **새로운 도전**: 아직 다루지 않은 토픽 중 우선순위가 높은 것
- **다음 단계**: 이전에 Level 1-2까지만 다룬 토픽의 심화 학습

### 5. 학습 패턴 분석
- 가장 많이 공부한 카테고리
- 가장 취약한 카테고리 (커버리지가 낮거나 난이도가 낮은 것)
- 학습 빈도 추세

## 출력 형식

깔끔하고 읽기 쉬운 마크다운 형식으로 출력하세요. 이모지를 적절히 활용하여 시각적으로 구분해주세요.

## 주의사항

- 이 모드에서는 소크라테스식 질문을 하지 않습니다
- 객관적인 데이터를 기반으로 현황을 보여주세요
- 추천은 강제가 아닌 제안입니다
- progress/log.md에 기록이 없으면 "아직 학습 기록이 없습니다. `/project:cs-learn`으로 첫 학습을 시작해보세요!"라고 안내하세요
